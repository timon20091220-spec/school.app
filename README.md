# 배문고 학교 앱 v5

관리자 로그인: `admin / 2026`

## 주요 기능
- 관리자 로그인 시 앱 재실행과 홈 버튼 모두 관리자 홈으로 이동
- 오늘의 배문 직접 수정
- 여러 축제 생성, 공개/숨김, 대표 축제 설정
- 축제별 체험 부스, 메뉴, 행사 일정 관리
- 부스 예약 시간을 비우면 즉시 예약, 입력하면 시간 선택 예약
- 공지 요약/상세/사진 및 수정·삭제
- 사진 클릭 전체 화면 보기
- 행사 알림 작성 및 알림함 누적, 알림 켜기/끄기

현재 데이터는 같은 기기의 브라우저 저장소에만 저장됩니다. 실제 전체 학생 공유는 Firebase 연결이 필요합니다.


## v6 업데이트
- 알림함의 `전체 삭제`를 관리 메뉴 방식으로 개선
- 모두 읽음, 읽은 알림 정리, 알림함 비우기 제공
- 알림함 비우기 전 별도 확인 화면 표시
- 오늘의 배문을 날짜별 예약 등록 방식으로 변경
- 시작·종료 시간을 기준으로 진행률을 30초마다 자동 갱신
- 날짜가 바뀌면 해당 날짜의 예약 일정으로 자동 전환
- 일정이 없는 날은 `주요 일정 없음`을 기본 표시
- 관리자는 미래 날짜의 오늘의 배문을 미리 등록·수정·삭제 가능
- 신규 축제의 체험 부스, 먹거리, 행사 일정은 빈 상태로 시작
- 이전 v5의 데모 부스·메뉴·일정은 첫 실행 시 자동 제거


## v7 학생 커뮤니티 및 첫 진입 화면
- 앱 첫 화면: 학생 로그인 / 게스트 입장 / 관리자 로그인
- 학생 로그인: 1~3학년, 1~10반, 1~30번, 이름, 비밀번호
- 시제품 최초 학생 비밀번호: `baemoon2026`
- 최초 로그인 후 개인 비밀번호 변경 화면
- 시제품 관리자: `admin / rlarkdals`
- 게스트 입장 확인창 및 관리자 공개 범위 설정
- 학생 커뮤니티: 게시글, 사진, 좋아요, 댓글, 신고
- 관리자 커뮤니티 관리: 실제 작성자 확인, 고정, 숨김, 삭제
- 관리자 학생 로그인 현황: 계정 정지/복구, 비밀번호 초기화

중요: 현재 계정과 글은 브라우저 localStorage에 저장되는 시제품입니다. 정식 운영에서는 학생 명단, Firebase Authentication, Firestore 보안 규칙, 서버 세션 검사가 필요합니다.


## v8 수정
- 학생 및 관리자 로그인 성공 후 환영 화면이 남는 CSS 오류 수정
- 관리자 로그인 `admin / rlarkdals` 정상 작동
- 관리자 로그인에서 Enter 키 지원
- 시제품 학생 로그인은 공통 비밀번호 `baemoon2026`으로 바로 입장
- 최초 비밀번호 변경 화면 비활성화
- 커뮤니티 익명 작성 옵션 제거
- 게시글과 댓글에 학년·반·번호·이름 표시
- 기존 익명 게시글과 댓글도 가능한 범위에서 실명 형식으로 변환


## v9 Firebase 연결 베타
- Firebase Authentication 학생·관리자·게스트 로그인
- Firestore 학생 프로필 저장
- 여러 기기에서 공유되는 실명 커뮤니티
- 게시글·댓글·좋아요·신고 실시간 동기화
- 관리자 커뮤니티 관리 및 학생 계정 정지
- `firestore.rules` 및 설정 안내 포함

**중요:** 업로드 후 Firebase 콘솔의 Firestore 규칙 탭에 `firestore.rules` 내용을 게시해야 합니다.
