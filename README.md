# 배문고 학교 앱 v5

관리자 로그인: `admin / 2026`

## 주요 기능
- 관리자 로그인 시 앱 재실행과 홈 버튼 모두 관리자 홈으로 이동
- 오늘의 배문 직접 수정
- 여러 축제 생성, 공개/숨김, 대표 축제 설정
- 축제별 체험 부스, 메뉴, 행사 일정 관리
- 부스 예약 시간을 비우면 즉시 예약, 입력하면 시간 선택 예약
- 공지 요약/상세/사진 및 수정·삭제
- 사진 클릭 전체 화면 보기
- 행사 알림 작성 및 알림함 누적, 알림 켜기/끄기

현재 데이터는 같은 기기의 브라우저 저장소에만 저장됩니다. 실제 전체 학생 공유는 Firebase 연결이 필요합니다.


## v6 업데이트
- 알림함의 `전체 삭제`를 관리 메뉴 방식으로 개선
- 모두 읽음, 읽은 알림 정리, 알림함 비우기 제공
- 알림함 비우기 전 별도 확인 화면 표시
- 오늘의 배문을 날짜별 예약 등록 방식으로 변경
- 시작·종료 시간을 기준으로 진행률을 30초마다 자동 갱신
- 날짜가 바뀌면 해당 날짜의 예약 일정으로 자동 전환
- 일정이 없는 날은 `주요 일정 없음`을 기본 표시
- 관리자는 미래 날짜의 오늘의 배문을 미리 등록·수정·삭제 가능
- 신규 축제의 체험 부스, 먹거리, 행사 일정은 빈 상태로 시작
- 이전 v5의 데모 부스·메뉴·일정은 첫 실행 시 자동 제거


## v7 학생 커뮤니티 및 첫 진입 화면
- 앱 첫 화면: 학생 로그인 / 게스트 입장 / 관리자 로그인
- 학생 로그인: 1~3학년, 1~10반, 1~30번, 이름, 비밀번호
- 시제품 최초 학생 비밀번호: `baemoon2026`
- 최초 로그인 후 개인 비밀번호 변경 화면
- 시제품 관리자: `admin / rlarkdals`
- 게스트 입장 확인창 및 관리자 공개 범위 설정
- 학생 커뮤니티: 게시글, 사진, 좋아요, 댓글, 신고
- 관리자 커뮤니티 관리: 실제 작성자 확인, 고정, 숨김, 삭제
- 관리자 학생 로그인 현황: 계정 정지/복구, 비밀번호 초기화

중요: 현재 계정과 글은 브라우저 localStorage에 저장되는 시제품입니다. 정식 운영에서는 학생 명단, Firebase Authentication, Firestore 보안 규칙, 서버 세션 검사가 필요합니다.

- 관리자 로그인 `admin / rlarkdals` 정상 작동
- 관리자 로그인에서 Enter 키 지원
- 시제품 학생 로그인은 공통 비밀번호 `baemoon2026`으로 바로 입장
- 최초 비밀번호 변경 화면 비활성화
- 커뮤니티 익명 작성 옵션 제거
- 게시글과 댓글에 학년·반·번호·이름 표시
- 기존 익명 게시글과 댓글도 가능한 범위에서 실명 형식으로 변환


## v9 Firebase 연결 베타
- Firebase Authentication 학생·관리자·게스트 로그인
- Firestore 학생 프로필 저장
- 여러 기기에서 공유되는 실명 커뮤니티
- 게시글·댓글·좋아요·신고 실시간 동기화
- 관리자 커뮤니티 관리 및 학생 계정 정지
- `firestore.rules` 및 설정 안내 포함

**중요:** 업로드 후 Firebase 콘솔의 Firestore 규칙 탭에 `firestore.rules` 내용을 게시해야 합니다.


## v10 Firebase 축제·급식 동기화
- 관리자가 수정한 축제 기본 정보, 부스, 먹거리, 행사 일정이 Firestore에 저장됩니다.
- 다른 휴대전화와 PC에 축제 변경 내용이 실시간 반영됩니다.
- v9에서 관리자 기기에 저장했던 기존 축제는 첫 관리자 로그인 시 자동으로 Firestore로 이전됩니다.
- 홈 화면에 오늘의 급식 카드가 추가되었습니다.
- 관리자는 미래 날짜의 급식을 미리 등록·수정·삭제할 수 있습니다.
- 급식 데이터도 모든 기기에 실시간 공유됩니다.

**업데이트된 `firestore.rules`를 Firebase 콘솔에 다시 게시해야 합니다.**


## v10.1 연결 점검 수정
- Firebase 서버 저장이 완료된 뒤에만 성공 메시지가 표시됩니다.
- 관리자 홈에 Firebase 연결 상태와 `연결 점검` 버튼이 추가되었습니다.
- 규칙 오류, 인터넷 오류, 문서 용량 초과를 화면에 표시합니다.
- 기존 축제 이전 실패가 관리자 로그인을 막지 않도록 수정했습니다.
- 축제나 급식 저장 실패 시 로컬 화면만 바뀌는 문제를 줄였습니다.


## v10.2 관리자 동기화 수정
- 관리자 커뮤니티에서 Firestore 게시글을 직접 새로고침할 수 있습니다.
- 커뮤니티 쿼리를 단순 컬렉션 리스너로 변경해 일부 문서가 누락되는 문제를 줄였습니다.
- 학생 기기의 기존 localStorage 게시글을 로그인 시 Firestore로 자동 이전합니다.
- Firebase 모듈이 로드되지 않았을 때 로컬 로그인·로컬 커뮤니티가 가짜로 작동하지 않도록 차단했습니다.
- 관리자 홈에 `전체 저장` 버튼을 추가했습니다.
- 전체 저장 대상: 축제, 급식, 공지, 오늘의 배문, 게스트 공개 설정
- 앱 설치 버튼은 파일 다운로드가 아니라 PWA 설치 기능임을 명확히 안내합니다.


## v11 전체 Firebase 통합
- Firebase Authentication: 학생, 관리자, 게스트 로그인
- Firestore 공용 데이터: 축제, 부스, 먹거리, 행사, 급식, 공지, 오늘의 배문, 게스트 공개 설정
- Firestore 커뮤니티: 게시글, 댓글, 좋아요, 신고, 관리자 숨김·고정·삭제
- Firestore 예약: 학생별 예약, 다른 기기 내 예약 확인, 관리자 전체 명단 및 CSV
- Firestore 알림: 관리자 전체/학년 알림, 학생 개인 행사 알림, 계정별 읽음·숨김 상태, 계정별 ON/OFF
- Firestore media: 공지·부스·메뉴·커뮤니티 사진을 별도 문서로 분리
- 기존 기기 localStorage 데이터 자동 이전
- 관리자 전체 저장 시 백업 기록 생성

공유·개인 데이터는 Firestore가 기준이며 localStorage는 로그인 화면 복구 및 구버전 이전용 보조 데이터로만 남습니다.


## v11.3 예약 권한 수정
- 신규 예약 전 중복 여부를 확인하는 단일 문서 조회 권한을 추가했습니다.
- 학생은 본인 UID로 시작하는 예약 문서만 단일 조회할 수 있습니다.
- 내 예약 목록은 `userUid == 로그인 UID` 조건으로만 조회됩니다.
- 예약 권한 오류가 다른 Firestore 오류와 구분되어 표시됩니다.


## v11.4 계정 관리 정리
- Firestore `users` 컬렉션이 비어 사라져 보이는 경우를 정상 상태로 안내합니다.
- 학생이 다시 로그인하면 누락된 `users/{uid}` 문서가 자동 생성됩니다.
- 관리자 학생 목록에 새로고침 버튼을 추가했습니다.
- 비밀번호 초기화 버튼을 실제 변경처럼 보이지 않도록 `초기화 안내`로 변경했습니다.
- 계정 삭제, 학생 문서 삭제, 계정 정지의 차이를 안내하는 팝업을 추가했습니다.
- v11.3 예약 권한 수정이 그대로 포함됩니다.


## v11.5 앱 설치 버튼 복원
- 안내창을 제거했습니다.
- 처음 버전처럼 상단 설치 버튼을 누르면 브라우저의 기본 앱 설치창이 바로 열립니다.
- 이미 설치된 경우 중복 설치를 막습니다.
- 설치 지원 브라우저에서 `beforeinstallprompt`가 준비될 때까지 짧게 기다린 뒤 실행합니다.
